# 性能测试报告
本文档展示了 AIDB 在向量检索场景下的性能表现。测试采用行业标准基准测试工具，在 XS 规格（4 credit）配置下进行。

## 测试方法
使用 [VectorDBBench](https://github.com/zilliztech/VectorDBBench) 进行向量检索性能评估。VectorDBBench 是业界广泛使用的向量数据库基准测试工具，能够全面评估向量数据库的检索性能。

## 测试配置
- 数据集规模：1000 万条向量数据（768 维）
- 实例规格：XS（4 credit）
- 索引参数：
 - ○vectors.hnsw_ef_search=200
 - ○vectors.max_scan_points=5000

## 测试结果
- QPS：2,808 次/秒
- 召回率（Recall）：93.2%
- P99 延迟：6.8 毫秒

**性能对比** 

下表展示了 AIDB 与其他主流向量数据库在相同测试场景下的性能对比：

| 产品 | P99 延迟（毫秒） | QPS | 召回率 |
|---|---:|---:|---:|
| **AIDB** | **6.8** | **2,808** | **0.932** |
| Milvus-16c64g-sq8 | 5.1 | 437 | 0.951 |
| QdrantCloud-16c64g | 9.2 | 446 | 0.936 |
| OpenSearch-16c128g-force_merge | 10.8 | 1,557 | 0.924 |
| ElasticCloud-8c60g-force_merge | 12.5 | 1,520 | 0.903 |
| Pinecone-p2.x8-1node | 14.1 | 1,131 | 0.902 |
| ElasticCloud-8c60g | 14.5 | 376 | 0.904 |
| OpenSearch-16c128g | 20.7 | 505 | 0.907 |

在千万级向量数据集上，AIDB 实现了高吞吐量和低延迟的平衡，召回率达到 **93.2%**，满足生产环境对向量检索的准确性和性能要求。从对比数据可以看出，AIDB 在实现最高 QPS 的同时，保持了较低的 P99 延迟和较高的召回率。

## 总结
AIDB 在 XS 规格（4 credit）配置下，展现出卓越的性能表现。
向量检索场景下，支持千万级向量数据的高性能检索，QPS 达到 2,808，P99 延迟低至 6.8 毫秒。
